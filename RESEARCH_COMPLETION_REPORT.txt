KEY FINDINGS
================================================================================

MOVE 2024 EDITION:
- Shared objects use `key` ability for standalone, `key,store` for wrappable
- Display standard uses string interpolation {field_name} for dynamic rendering
- Error codes should group by feature (1-10 auth, 101-110 logic, 201-210 sponsor)
- Version field pattern optional but recommended for replay attack prevention

NEXT.JS 16 + REACT 19:
- Server components are default; only add "use client" when needed
- Route groups (dapp)/ isolate providers, keeping landing page bundle small
- useActionState handles transaction form state (pending/error/success)
- Dynamic imports with ssr:false critical for dApp-kit (500KB+)

DAPP-KIT INTEGRATION:
- useSignAndExecuteTransaction main hook; returns {mutate, isPending, error}
- Sponsored transactions: user signs → API sponsors → executeBlock with both
- useSuiClientQuery with showContent:true fetches shared objects
- Type-safe parsing prevents "Cannot read property of undefined" errors

PRODUCTION OPTIMIZATION:
- Dynamic imports reduce dApp-kit load by ~500KB initially
- QR scanner only loads on /scan page, not landing page
- Tailwind final CSS should be <50KB; check content paths carefully
- Route group isolation ensures landing page doesn't load wallet context

================================================================================
CRITICAL GOTCHAS TO AVOID
================================================================================

1. Don't use "use client" at root level → Be granular, isolate to components
2. Don't skip transaction error handling → "object locked" needs exponential backoff
3. Don't publish Display with wrong visibility → Only Publisher can modify templates
4. Don't forget showContent:true → useSuiClientQuery returns empty objects without it
5. Don't bundle dApp-kit in landing page → Use dynamic imports + route groups
6. Don't disable TypeScript strict mode → Catches object parsing errors early
7. Don't use dynamic CSS class names → Tailwind purge can't find concatenated classes
